DevOps

As you have seen in the video above, I do have a good number of projects in Azure DevOps. I have a small team of developers that work with me on most projects. In the projects, I lead the team in design and architecture. I use the Azure Boards for a High level work flow and we drill down to the work item. I do create the backlogs for the group and assign work items to the other developers. We meet at least once a week to discuss progress and to determine if they need extra help. I also actively develop with the work items assign to myself. In an effort to become more agile and productive as possible, I have created a build pipeline and a deployment pipeline. When a developer submits a code to the repository, once it is approved, that codebase commit triggers a build in the build pipeline. Once the build pipeline is done, the project goes into the release pipeline and a series of task are executed an pushed out into the Azure Web Service account.

Cloud Infrastructure

As we bring things projects online, we will have to first build out the Cloud Infrastructure. The choice to use a particular cloud infrastructure for example, AWS or Azure, depends on a few key items. First, we look at the project itself and determine if the project runs better on Azure or AWS. We have just finished the deployment plan for 2 projects and we have determined that we will be using both AWS and Azure in a small capacity at first and as we complete the configuration phase of the Cloud Infrastructure, we will be able to deploy to both platforms.
